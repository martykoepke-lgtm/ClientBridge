The Engine That Makes This Special
Three systems working as one brain — spaced repetition, prerequisite regression, and a contextually aware AI tutor. No competitor combines all three.
In plain English, here is what happens:

A student is working through a lesson on heart failure. They answer an MCQ about why BNP is elevated — and get it wrong. The system now does three things simultaneously:

1
Spaced repetition brings it back sooner
That BNP card was scheduled for review in 7 days. Now it's coming back tomorrow. The system shortens the interval because the student demonstrated they haven't mastered it yet.
2
The system checks: why did they get it wrong?
Heart failure has prerequisites — you need to understand cardiac output to understand why BNP rises. The system looks at the student's scores on the prerequisite concepts. If cardiac output is weak (say, 52%), the system surfaces those foundational cards first — before pushing forward to harder material. It takes you backward to the building block that's shaky, so you can move forward on solid ground.
3
The AI tutor knows all of this when it talks to the student
When the student opens the Socratic tutor for heart failure, the AI doesn't just know the lesson content — it knows "this learner got BNP wrong, their cardiac output foundation is at 52%, and they mastered Frank-Starling last week at 88%." So instead of asking a generic question, it says: "Let's go back to basics — walk me through how cardiac output is calculated, and then we'll connect that to why BNP rises in heart failure." It meets the student exactly where they are.

AI Agents — Who Does What
Five distinct AI agents power the platform. Each has a specific role, defined scope, and hard boundaries. They never overlap. Here is each one.
AGENT 1 — BACKSTAGE
Content Generator
Phase 1
Role
Takes Dr. Scariati's micromodule framework + NBOME taxonomy and generates complete lesson drafts in bulk batches.
Scope
Generates: clinical hook, core concept, supporting details (max 3), learning objectives, flashcards (front/back), AI tutor system prompt per lesson.
Boundaries
Never sees student data. Never generates student-facing responses. Output is always a structured JSON draft that goes through QA before any human sees it.
// Input
Framework doc + Domain: "Cardiovascular" + Subdomain: "Heart Failure"
Bloom's target: L2 (Understand) + NBOME objectives for this topic

// Output (JSON)
{ clinical_hook, core_concept, supporting_details[3],
  flashcards[{front, back}], ai_tutor_prompt,
  comlex_high_yield: true/false, osteopathic_integration: true/false }
AGENT 2 — BACKSTAGE
MCQ Builder
Phase 1
Role
Generates board-style MCQs in a three-step process: clinical vignette → question + 4-5 answer options → full explanations for every option.
Scope
80-120 word vignettes. One correct answer. Plausible distractors representing real misconceptions. Why-right and why-wrong for each option. COMLEX high-yield flag.
Boundaries
Must follow NBOME format exactly. No ambiguous questions. No "all of the above." Each question is Bloom's-tagged. Output is structured JSON — never free text.
Step 1: Vignette → Step 2: Q + Answers → Step 3: Explanations
AGENT 3 — BACKSTAGE
QA Validator
Phase 1
Role
Runs automated quality checks on every generated lesson before any human sees it. Gatekeeper between AI output and SME review.
Scope
6 checks: single learning goal, ≤3 concepts, required sections present, valid JSON schema, no AI artifacts ("as an AI..."), minimum content length.
Boundaries
Does NOT evaluate medical accuracy — that's the SME's job. Only checks structure and format. Failed lessons retry once automatically. Second failure → human review.
Pass → SME queue Retry once → regenerate Fail twice → human attention
AGENT 4 — STUDENT-FACING
Socratic Tutor
Phase 2
Role
Conducts guided Socratic dialogue with students, pushing them toward L5-L6 reasoning. Never gives answers — asks questions that help students discover connections themselves.
Scope
Receives: lesson content (Layer 3) + anonymous learner context (Layer 4: scores, weak prerequisites, session history). Knows where the student is in their learning journey.
Boundaries
FERPA: Never receives student name, ID, or school.
Scope: Stays within current module — won't discuss unrelated topics.
Identity: Will not reveal who the student is, even if asked.
Method: Socratic only — questions, not lectures.
// 4-layer system prompt
Layer 1 (static): Identity refusal + safety rules
Layer 2 (static): Socratic behavior — ask, don't tell
Layer 3 (per lesson): Lesson content, objectives, clinical hook
Layer 4 (dynamic): "This learner scored 38% on HF MCQs,
  prerequisite 'cardiac output' is weak at 52%,
  'Frank-Starling' is strong at 88%"
This is the only agent that talks to students. All other agents work behind the scenes in the Backstage. The Socratic Tutor is the only AI that has a real-time conversation with a human learner.
AGENT 5 — BACKGROUND
Session Evaluator
Phase 2
Role
Runs after each AI tutor session ends. Evaluates the conversation against the lesson's learning objectives and Bloom's target level. Returns a structured score.
Scope
Input: full session transcript + lesson objectives. Output: JSON only — Bloom's level achieved, score (0-100), weak areas array, feedback summary, XP earned.
Boundaries
Never talks to students. Runs server-side after session closes. Output feeds into spaced repetition engine and prerequisite scoring. No free-text responses — JSON only.
// Output (JSON only)
{
  blooms_level_achieved: 3,
  score: 72,
  weak_areas: ["cardiac output calculation", "BNP trigger"],
  feedback_summary: "Student understands Frank-Starling but
    cannot connect reduced CO to compensatory mechanisms",
  xp_earned: 15
}
